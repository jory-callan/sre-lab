## Skill 书写技巧

### 技术写作原则
- 明确的受众定位
- 结构化的内容组织
- 示例驱动的讲解
- 预见用户的问题

### 编写建议
1. **从实际问题出发** - 解决真实遇到的痛点
2. **先操作再抽象** - 先写出步骤，再提炼模式
3. **包含正反案例** - 说明怎么做和不怎么做
4. **提供验证方法** - 让用户知道做对了没有

## 贡献指南

### 如何为本项目贡献 Skill

1. 阅读 [SKILL_WRITING_SOP.md](./SKILL_WRITING_SOP.md)
2. 在合适的分类目录下创建 Skill
3. 提交 Pull Request 或直接提交（根据团队规范）
4. 更新相关文档

### Skill 审核标准
- 解决实际问题
- 步骤清晰可执行
- 包含验证方法
- 考虑异常情况
