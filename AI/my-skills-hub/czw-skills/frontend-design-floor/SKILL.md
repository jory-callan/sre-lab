# 前端设计保底方案（高下限）

当用户简单描述需求时，按以下步骤处理：

## 1. 自动选择设计参考风格（默认按需求类型）

| 需求类型 | 默认风格 | 理由 |
|---------|---------|------|
| 管理后台/Dashboard | Linear 风格 | 极简深色，精密，数据友好 |
| 营销/官网 | Stripe 风格 | 优雅紫渐变，高端信任感 |
| 开发者工具 | Vercel 风格 | 黑白简约，精准专业 |
| 内容类应用 | Notion 风格 | 温暖极简，易读舒适 |
| **默认** | **Linear 风格** | **最稳妥的现代设计** |

## 2. 自动应用设计原则

### ✅ 必须做的
- 先建立完整设计系统，再写页面组件
- 使用 CSS 变量定义（颜色、字体、间距、阴影、圆角）
- 确保清晰的视觉层次（字体大小、粗细、颜色对比度）
- 使用现代 CSS（Grid、Flexbox、container queries）
- 完整的交互状态（hover、focus、active、loading、error、success）
- 响应式设计（mobile-first）
- 尊重 `prefers-reduced-motion`
- 可访问性（点击目标 ≥44px，对比度符合 WCAG）

### ❌ 必须避免的反模式
- 激进渐变背景
- 默认 glassmorphism
- 到处都是 emoji
- 通用 SaaS 卡片堆满图标
- 左边界强调卡片
- 假数据填充的仪表板
- 过度装饰的 SVG 插图
- 超大圆角作为层级替代
- 彩虹调色板
- 模糊标签如"洞察""增长""扩展""优化"

## 3. 每次写前端时必须加载的技能

- `claude-design` - 设计流程和品味
- `popular-web-designs` - 选择对应风格模板

### 参考文件
- `references/linear-design-system.md` - Linear 风格快速参考（颜色、字体、组件模板）

## 4. 用户简单描述时的标准处理流程

### 情况 A：用户说"做一个像 X 产品那样的界面"
1. 识别 X 产品的类型（管理后台/营销页/开发者工具...）
2. 从 `popular-web-designs` 中选择最匹配的风格（Linear/Stripe/Vercel...）
3. 加载对应风格模板，提取设计系统
4. 规划功能模块（参考 X 产品的核心元素）
5. 与用户确认功能范围和输出格式
6. 实现时严格遵循所选设计系统

### 情况 B：用户只说"做一个 Y 功能界面"
1. 根据 Y 功能类型自动选择默认风格
2. 建立完整设计系统（CSS variables）
3. 实现页面，确保所有交互状态
4. 如果需要探索，给出三个方向：
   - **保守** - 最接近现有模式，风险最低
   - **强适配** - 对需求的最佳诠释
   - **发散** - 更创新，用于发现品味边界

## 5. 技术栈（按用户偏好）

- 前端：React + shadcn/ui
- 后端：Go + Echo + GORM + Viper

## 使用示例

### 简单需求 → 高下限结果

**用户说**："写一个登录页面"

**我的处理**：
1. 选择 Linear 风格（默认）
2. 加载 `popular-web-designs/templates/linear.app.md`
3. 建立设计系统
4. 写完整的登录页面，包含所有交互状态

**用户说**："做一个管理后台"

**我的处理**：
1. 选择 Linear 风格（管理后台默认）
2. 加载对应模板
3. 建立设计系统
4. 写完整的 Dashboard
