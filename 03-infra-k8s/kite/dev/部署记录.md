# Kite 可视化管理工具部署记录
**日期**：2026-05-30
**项目地址**：https://github.com/kite-org/kite
**集群**：K3s 三节点 HA

## 概述
Kite 是一个轻量级、现代化的 Kubernetes 仪表板，提供：
- 多集群管理
- 实时监控（CPU、内存、网络）
- 资源管理（Pod、Deployment、Service 等）
- 多语言支持（中文/英文）
- Web 终端、AI 助手等

---

## 部署前准备
### 1. 本机配置
- 复制集群 kubeconfig 到本机
- 修改 server 地址为 192.168.5.249
- 验证集群连接：`kubectl get nodes`

### 2. 准备文件
从 GitHub 获取 Kite 项目 deploy 文件夹中的 install.yaml

---

## 部署步骤
### 1. 创建自定义部署文件 kite.yaml
- Namespace: kite
- ServiceAccount + ClusterRoleBinding (cluster-admin)
- Deployment: ghcr.io/kite-org/kite:latest
- Service: NodePort (30080)
- Ingress: traefik ingress (可选)

### 2. 部署
```bash
kubectl apply -f kite.yaml
```

### 3. 验证
```bash
kubectl get pods -n kite
kubectl get svc -n kite
```

## 部署文件
### kite.yaml
```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: kite
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: kite
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
  - kind: ServiceAccount
    name: kite
    namespace: kite
---
kind: ServiceAccount
apiVersion: v1
metadata:
  name: kite
  namespace: kite
---
apiVersion: apps/v1
kind: Deployment
metadata:
  labels:
    app: kite
  name: kite
  namespace: kite
spec:
  replicas: 1
  selector:
    matchLabels:
      app: kite
  template:
    metadata:
      labels:
        app: kite
    spec:
      serviceAccountName: kite
      containers:
        - image: ghcr.io/kite-org/kite:latest
          imagePullPolicy: IfNotPresent
          name: kite
          resources:
            limits:
              cpu: 1000m
              memory: 512Mi
            requests:
              cpu: 100m
              memory: 128Mi
          livenessProbe:
            httpGet:
              path: /healthz
              port: 8080
            initialDelaySeconds: 10
            periodSeconds: 10
          readinessProbe:
            httpGet:
              path: /healthz
              port: 8080
            initialDelaySeconds: 5
            periodSeconds: 5
          ports:
            - containerPort: 8080
              protocol: TCP
---
apiVersion: v1
kind: Service
metadata:
  labels:
    app: kite
  name: kite
  namespace: kite
spec:
  ports:
    - name: http
      port: 8080
      protocol: TCP
      targetPort: 8080
      nodePort: 30080
  selector:
    app: kite
  type: NodePort
---
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: kite
  namespace: kite
spec:
  ingressClassName: traefik
  rules:
    - host: kite.local
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: kite
                port:
                  number: 8080
```

## 常用命令
### 查看日志
```bash
kubectl logs -n kite -l app=kite -f
```

### 升级
```bash
# 修改 image 标签后重新 apply
kubectl apply -f kite.yaml
```

### 卸载
```bash
kubectl delete -f kite.yaml
```

---

## 注意事项
1. 镜像 ghcr.io/kite-org/kite:latest 通过 Nexus3 镜像仓库拉取（配置在 registries.yaml）
2. 默认使用 cluster-admin 权限，生产环境建议配置更细粒度的 RBAC
3. 可以配置 Ingress 或 LoadBalancer 方便访问
