# 踩坑记录

## 1. Operator 生成密码逻辑

### 现象

未预创建 Secret 时，operator 自动生成随机密码，且不显示在日志中。
需要手动从 Secret 中提取密码：

```bash
kubectl get secret mysql-standalone-secrets -n mysql -o jsonpath='{.data.root}' | base64 -d
```

### 解决

预创建 `mysql-cluster-secrets` 指定已知密码，并在 CR 中通过 `secretsName` 引用：

```yaml
spec:
  mysql:
    secretsName: mysql-cluster-secrets
```

## 2. Standalone 和 Cluster 互斥

### 说明

同一命名空间中只能存在一个 PerconaServerMySQL CR，
因为 operator 的资源命名规则（Service、StatefulSet 等）会冲突。

### 解决方法

切换模式前先卸载旧的：
```bash
./uninstall.sh standalone   # 或 cluster
./install.sh cluster        # 安装新模式
```

## 3. Percona 镜像地址

### 说明

Percona 官方镜像托管在 Docker Hub 的 `percona/` 命名空间下：
- `percona/percona-server:8.4.8-8.1` — MySQL 8.4 镜像
- `percona/haproxy:2.8.18-1` — HAProxy 代理
- `percona/percona-mysql-router:8.4.8` — MySQL Router
- `percona/percona-xtrabackup:8.4.0-5.1` — 备份工具

如果 k3s 配置了 mirror，直接使用原始镜像名即可。
