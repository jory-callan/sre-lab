## 3. 部署步骤（K3s）

### 3.1 安装第一个节点 (192.168.5.249)

#### 分发配置文件
```bash
for ip in 192.168.5.249 192.168.5.101 192.168.5.100; do
  ssh root@$ip 'mkdir -p /etc/rancher/k3s'
  scp config.yaml registries.yaml root@$ip:/etc/rancher/k3s/
done
```

#### 执行安装
使用 rancher 国内镜像：
```bash
ssh root@192.168.5.249 '
  curl -sfL https://rancher-mirror.rancher.cn/k3s/k3s-install.sh | \
  INSTALL_K3S_VERSION=v1.31.5-k3s1 INSTALL_K3S_MIRROR=cn \
  sh -s - server --cluster-init --config /etc/rancher/k3s/config.yaml
'
```

#### 验证
```bash
ssh root@192.168.5.249 'k3s kubectl get nodes'
```

### 3.2 获取 Token
```bash
TOKEN=$(ssh root@192.168.5.249 'cat /var/lib/rancher/k3s/server/token')
echo $TOKEN
```

### 3.3 加入第二节点 (192.168.5.101)
```bash
ssh root@192.168.5.101 '
  curl -sfL https://rancher-mirror.rancher.cn/k3s/k3s-install.sh | \
  INSTALL_K3S_VERSION=v1.31.5-k3s1 INSTALL_K3S_MIRROR=cn \
  sh -s - server --server https://192.168.5.249:6443 --token $TOKEN --config /etc/rancher/k3s/config.yaml
'
```

### 3.4 加入第三节点 (192.168.5.100)
```bash
ssh root@192.168.5.100 '
  curl -sfL https://rancher-mirror.rancher.cn/k3s/k3s-install.sh | \
  INSTALL_K3S_VERSION=v1.31.5-k3s1 INSTALL_K3S_MIRROR=cn \
  sh -s - server --server https://192.168.5.249:6443 --token $TOKEN --config /etc/rancher/k3s/config.yaml
```

### 3.5 获取 kubeconfig 到本地

## 5. 问题与解决方案

### 问题 1：etcd 参数配置错误
**错误信息**：
```
Error: unknown flag: --auto-compaction-retention
```
**原因**：config.yaml 中 etcd 参数配置格式错误
**解决**：简化 config.yaml，移除自定义 etcd 参数，使用默认配置

---

### 问题 2：kube-apiserver 参数错误
**错误信息**：
```
Error: unknown flag: --kube-api-burst
```
**原因**：kube-apiserver 没有 --kube-api-burst 参数
**解决**：移除 config.yaml 中的 kube-apiserver-arg、kube-controller-manager-arg、kube-scheduler-arg 的自定义配置

---

### 问题 3：主机名冲突
**错误信息**：
```
etcd cluster join failed: duplicate node name found
```
**原因**：三台机器初始主机名都是 k3s-server-1
**解决**：修改主机名分别为 k3s-server-1、k3s-server-2、k3s-server-3

---

### 问题 4：GitHub 访问失败
**解决**：使用 gh-proxy.com 加速 GitHub 资源访问

---

### 问题 5：k3s 官方安装脚本访问失败
**解决**：使用 rancher 国内镜像：https://rancher-mirror.rancher.cn/k3s/k3s-install.sh

---

### 问题 6：*.local 域名在 macOS 上解析延迟 5 秒
**原因**：macOS 对 *.local 有特殊的 Bonjour/mDNS 处理
**解决**：统一使用 *.czw-sre.internal 域名

## 10. 注意事项

### 国内部署
1. 使用 rancher 国内镜像安装脚本
2. 使用 gh-proxy 加速 GitHub 资源
3. 配置国内镜像仓库
4. 避免使用 *.local 域名，统一使用 *.czw-sre.internal

### 生产环境
1. 建议使用独立数据盘（挂载到 /var/lib/rancher/k3s）
2. 配置资源限制
3. 配置备份策略（etcd 快照）
4. 配置监控和告警
5. 配置日志轮转

### MetalLB 说明
1. L2 模式下 IP 不响应 ping 是正常的（只宣告 ARP，响应 TCP/UDP）
2. IP 由任意一个 speaker 节点宣告，通过 ARP 选举决定
