## 2. L2 模式详解

### 2.1 工作原理
- L2 模式下，MetalLB 从 IP 池中分配 IP 给 LoadBalancer Service
- 某一个节点（speaker）宣告该 IP 的 ARP 响应
- 客户端通过 ARP 找到该节点，流量直接路由到该节点
- 该节点再通过 kube-proxy 将流量转发到后端 Pod

### 2.2 基本 L2 配置
```yaml
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: default
  namespace: metallb-system
spec:
  addresses:
  - 192.168.5.240-192.168.5.250
  # 或者使用 CIDR
  # - 192.168.5.240/28
  autoAssign: true
  avoidBuggyIPs: true

---
apiVersion: metallb.io/v1beta1
kind: L2Advertisement
metadata:
  name: default
  namespace: metallb-system
spec:
  ipAddressPools:
  - default
```

### 2.3 高级 L2 配置

#### 指定节点宣告
```yaml
apiVersion: metallb.io/v1beta1
kind: L2Advertisement
metadata:
  name: default
  namespace: metallb-system
spec:
  ipAddressPools:
  - default
  nodeSelectors:
  - matchLabels:
      kubernetes.io/hostname: k3s-server-1
```

#### 多个 IP 池
```yaml
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: pool-a
  namespace: metallb-system
spec:
  addresses:
  - 192.168.5.240-192.168.5.245

---
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: pool-b
  namespace: metallb-system
spec:
  addresses:
  - 192.168.5.246-192.168.5.250

---
apiVersion: metallb.io/v1beta1
kind: L2Advertisement
metadata:
  name: all-pools
  namespace: metallb-system
spec:
  ipAddressPools:
  - pool-a
  - pool-b
```

#### Service 指定 IP 池
```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app
  annotations:
    metallb.universe.tf/address-pool: pool-a
spec:
  type: LoadBalancer
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 8080
```

#### Service 指定 IP
```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app
spec:
  type: LoadBalancer
  loadBalancerIP: 192.168.5.240
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 8080
```

### 2.4 L2 模式注意事项
1. **IP 不响应 ping**：这是正常的，MetalLB 只宣告 ARP，响应 TCP/UDP
2. **单节点瓶颈**：某个 IP 流量只会经过一个节点（该节点宣告 ARP）
3. **节点故障**：如果宣告节点故障，其他节点会接管（约 10 秒）
4. **IP 冲突**：确保 IP 池范围内的 IP 没有被其他设备占用

## 3. BGP 模式详解

### 3.1 工作原理
- MetalLB 通过 BGP 协议与路由器/交换机交换路由
- 每个 speaker 节点宣告 IP 池路由
- 路由器 ECMP（等价多路径）负载均衡流量到多个节点
- 适合高吞吐量场景，无单点瓶颈

### 3.2 BGP 配置示例

#### 基础 BGP 配置
```yaml
apiVersion: metallb.io/v1beta2
kind: BGPPeer
metadata:
  name: my-router
  namespace: metallb-system
spec:
  peerAddress: 192.168.5.1
  peerASN: 64501
  myASN: 64500

---
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: bgp-pool
  namespace: metallb-system
spec:
  addresses:
  - 192.168.10.0/24

---
apiVersion: metallb.io/v1beta1
kind: BGPAdvertisement
metadata:
  name: bgp-advertisement
  namespace: metallb-system
spec:
  ipAddressPools:
  - bgp-pool
```

#### 多 Peer 配置
```yaml
apiVersion: metallb.io/v1beta2
kind: BGPPeer
metadata:
  name: router-1
  namespace: metallb-system
spec:
  peerAddress: 192.168.5.1
  peerASN: 64501
  myASN: 64500

---
apiVersion: metallb.io/v1beta2
kind: BGPPeer
metadata:
  name: router-2
  namespace: metallb-system
spec:
  peerAddress: 192.168.5.2
  peerASN: 64502
  myASN: 64500
```

#### BGP 认证
```yaml
apiVersion: metallb.io/v1beta2
kind: BGPPeer
metadata:
  name: my-router
  namespace: metallb-system
spec:
  peerAddress: 192.168.5.1
  peerASN: 64501
  myASN: 64500
  password: "your-bgp-password"
```

#### BGP 节点选择
```yaml
apiVersion: metallb.io/v1beta2
kind: BGPPeer
metadata:
  name: my-router
  namespace: metallb-system
spec:
  peerAddress: 192.168.5.1
  peerASN: 64501
  myASN: 64500
  nodeSelectors:
  - matchLabels:
      node-role.kubernetes.io/control-plane: ""
```

## 4. IP 地址池配置

### 4.1 多种地址格式
```yaml
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: multi-format-pool
  namespace: metallb-system
spec:
  addresses:
  # 范围格式
  - 192.168.5.240-192.168.5.250
  # CIDR 格式
  - 192.168.10.0/28
  # 单个 IP
  - 192.168.20.100/32
```

### 4.2 自动分配控制
```yaml
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: manual-pool
  namespace: metallb-system
spec:
  addresses:
  - 192.168.5.240-192.168.5.250
  autoAssign: false  # 需要 Service 显式指定
```

### 4.3 避免 Buggy IP
```yaml
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: safe-pool
  namespace: metallb-system
spec:
  addresses:
  - 192.168.5.240-192.168.5.250
  avoidBuggyIPs: true  # 跳过 .0 和 .255
```

### 4.4 多 IP 池策略
```yaml
# 生产环境 IP 池
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: production-pool
  namespace: metallb-system
spec:
  addresses:
  - 192.168.5.240-192.168.5.245

---
# 测试环境 IP 池
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: staging-pool
  namespace: metallb-system
spec:
  addresses:
  - 192.168.5.246-192.168.5.250
```

使用时：
```yaml
apiVersion: v1
kind: Service
metadata:
  name: production-app
  annotations:
    metallb.universe.tf/address-pool: production-pool
spec:
  type: LoadBalancer
  selector:
    app: production-app
  ports:
  - port: 80
    targetPort: 8080
```

## 5. 高级特性

### 5.1 流量策略
```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app
spec:
  type: LoadBalancer
  externalTrafficPolicy: Local  # 或 Cluster
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 8080
```

- **Cluster**（默认）：流量可以转发到任意节点的 Pod，SNAT 会改变源 IP
- **Local**：流量只转发到本节点的 Pod，保留源 IP，但可能造成负载不均

### 5.2 健康检查
```yaml
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: monitored-pool
  namespace: metallb-system
spec:
  addresses:
  - 192.168.5.240-192.168.5.250

---
apiVersion: metallb.io/v1beta1
kind: BGPAdvertisement
metadata:
  name: monitored-advertisement
  namespace: metallb-system
spec:
  ipAddressPools:
  - monitored-pool
  # L2 模式不支持健康检查，BGP 模式支持
```

### 5.3 共享 IP
多个 Service 共享同一个 IP（不同端口）：

```yaml
apiVersion: v1
kind: Service
metadata:
  name: http-service
  annotations:
    metallb.universe.tf/allow-shared-ip: "shared-ip-key"
spec:
  type: LoadBalancer
  selector:
    app: http-app
  ports:
  - port: 80
    targetPort: 8080

---
apiVersion: v1
kind: Service
metadata:
  name: https-service
  annotations:
    metallb.universe.tf/allow-shared-ip: "shared-ip-key"
spec:
  type: LoadBalancer
  loadBalancerIP: 192.168.5.240
  selector:
    app: https-app
  ports:
  - port: 443
    targetPort: 8443
```

## 6. 故障排查

### 6.1 查看 Pod 状态
```bash
kubectl get pods -n metallb-system -o wide
```

### 6.2 查看 Controller 日志
```bash
kubectl logs -n metallb-system -l app=metallb,component=controller -f
```

### 6.3 查看 Speaker 日志
```bash
kubectl logs -n metallb-system -l app=metallb,component=speaker -f
```

### 6.4 查看资源状态
```bash
# 查看 IP 池
kubectl get ipaddresspools -n metallb-system
kubectl describe ipaddresspools -n metallb-system <name>

# 查看 L2 宣告
kubectl get l2advertisements -n metallb-system
kubectl describe l2advertisements -n metallb-system <name>

# 查看 BGP Peer（如果用 BGP 模式）
kubectl get bgppeers -n metallb-system
kubectl describe bgppeers -n metallb-system <name>

# 查看 BGP 宣告（如果用 BGP 模式）
kubectl get bgpadvertisements -n metallb-system
kubectl describe bgpadvertisements -n metallb-system <name>
```

### 6.5 常见问题

#### 问题 1：IP 分配不到
**症状**：Service 的 EXTERNAL-IP 一直是 pending
**排查**：
```bash
kubectl describe svc <service-name>
kubectl logs -n metallb-system -l app=metallb,component=controller
```

**可能原因**：
- IP 池未配置
- IP 池地址耗尽
- 自动分配关闭

#### 问题 2：IP 分配了但无法访问
**症状**：Service 有 EXTERNAL-IP，但无法访问
**排查**：
```bash
# 检查 speaker 日志
kubectl logs -n metallb-system -l app=metallb,component=speaker

# 检查 ARP（L2 模式）
arp -n | grep <external-ip>
```

**可能原因**：
- 节点防火墙问题
- IP 冲突
- 网络隔离

#### 问题 3：BGP 邻居无法建立
**症状**：BGP 模式下，路由无法宣告
**排查**：
```bash
kubectl logs -n metallb-system -l app=metallb,component=speaker
```

**可能原因**：
- ASN 配置错误
- BGP 密码错误
- 网络不通
- 路由器不接受连接

## 7. 常用命令

```bash
# 查看 Pod
kubectl get pods -n metallb-system -o wide

# 查看 IP 池
kubectl get ipaddresspools -n metallb-system

# 查看 IP 池详情
kubectl describe ipaddresspools -n metallb-system <name>

# 查看 L2 宣告
kubectl get l2advertisements -n metallb-system

# 查看 L2 宣告详情

# 查看 BGP Peer（BGP 模式）
kubectl get bgppeers -n metallb-system

kubectl get bgpadvertisements -n metallb-system

# 查看 Controller 日志
kubectl logs -n metallb-system -l app=metallb,component=controller -f

# 查看 Speaker 日志
kubectl logs -n metallb-system -l app=metallb,component=speaker -f

# 查看 Service
kubectl get svc -A

# 查看 Service 详情
kubectl describe svc -n <namespace> <name>

kubectl rollout restart deployment -n metallb-system controller
```
