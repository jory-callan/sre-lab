## Pod 问题

### 1. Pod 一直处于 ImagePullBackOff / ErrImagePull

**原因：**
- 镜像仓库无法访问
- 镜像名称/标签错误
- 私有镜像需要 credentials

**排查：**
```bash
kubectl describe pod <pod-name> -n temporal-simple
kubectl get events -n temporal-simple --sort-by='.lastTimestamp'
```

**解决：**
- 用集群中已有的镜像
- 或者配置 imagePullSecrets

---

### 2. Pod 一直 CrashLoopBackOff

**原因：**
- 配置错误
- 数据库连接失败
- 资源不足

**排查：**
```bash
kubectl logs <pod-name> -n temporal-simple --tail=100
kubectl describe pod <pod-name> -n temporal-simple
```

**解决：**
- 检查配置文件是否正确
- 检查数据库是否正常
- 增加资源 requests/limits

---

### 3. Pod 一直 Pending

**原因：**
- 资源不足
- 节点选择器不匹配
- 持久卷申请失败

**排查：**
```bash
kubectl describe pod <pod-name> -n temporal-simple
kubectl get pvc -n temporal-simple
kubectl get nodes
```

## Namespace 问题

### 1. Namespace 创建后 Retention 不是 1年

**原因：**
- 没用 wrapper 脚本
- 忘了加 `--retention` 参数

**解决：**
```bash
# 更新 Namespace
kubectl exec -n temporal-simple deploy/temporal-simple-admintools -- \
  temporal operator namespace update my-namespace \
  --address temporal-simple-frontend:7233 \
  --retention 8760h0m0s
```

---

### 2. Namespace 无法删除

**原因：**
- 有正在运行的 Workflow
- 有未完成的 Task

**解决：**
```bash
# 先终止所有 Workflow
kubectl exec -n temporal-simple deploy/temporal-simple-admintools -- \
  temporal workflow terminate --all \
  --address temporal-simple-frontend:7233 \
  --namespace my-namespace

# 再删除 Namespace
kubectl exec -n temporal-simple deploy/temporal-simple-admintools -- \
  temporal operator namespace delete my-namespace \
  --address temporal-simple-frontend:7233
```

## 连接问题

### 1. Web UI 无法访问

**原因：**
- 端口转发没启动
- Service 没有正确暴露
- Pod 没正常运行

**排查：**
```bash
kubectl get pods -n temporal-simple
kubectl get svc -n temporal-simple
kubectl port-forward -n temporal-simple svc/temporal-simple-web 8080:8080
```

---

### 2. Frontend 连接不上

**原因：**
- Server 没正常运行
- 端口转发没启动
- 地址错误

**排查：**
```bash
# 测试连接
kubectl exec -n temporal-simple deploy/temporal-simple-admintools -- \
  temporal operator namespace list --address temporal-simple-frontend:7233
```

## 日志查看

### 1. Server 日志

```bash
# 查看最后 100 行
kubectl logs -n temporal-simple deploy/temporal-simple-server --tail=100

# 实时跟踪日志
kubectl logs -n temporal-simple deploy/temporal-simple-server -f

# 查看前一个实例的日志
kubectl logs -n temporal-simple deploy/temporal-simple-server -p
```

---

### 2. 按级别查看日志

```bash
# 只看 ERROR
kubectl logs -n temporal-simple deploy/temporal-simple-server --tail=100 | grep -i error

# 只看 WARN
kubectl logs -n temporal-simple deploy/temporal-simple-server --tail=100 | grep -i warn
```

## 快速排查检查清单

部署后检查：

- [ ] 所有 Pod 是 Running 或 Completed
- [ ] Server 日志没有 ERROR
- [ ] Namespace Job 已完成
- [ ] Schema Job 已完成
- [ ] default Namespace Retention 是 8760h
- [ ] 能访问 Web UI
- [ ] admin-tools 能执行命令
