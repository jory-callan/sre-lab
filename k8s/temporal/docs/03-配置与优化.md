## 生产优化配置

### 1. Server 配置 (`config_template.yaml`)

```yaml
# 限流配置
history.ratelimit: 20000
frontend.ratelimit: 10000
matching.ratelimit: 10000
worker.ratelimit: 10000

# 持久化 QPS 限制
history.persistenceMaxQPS: 3000
history.persistenceIndividualNamespaceQPS: 1000
history.persistenceGlobalNamespaceQPS: 1000
matching.persistenceMaxQPS: 3000
worker.persistenceMaxQPS: 1000

# 任务队列优化
matching.numTaskqueueReadPartitions: 10
matching.numTaskqueueWritePartitions: 10
matching.numTaskqueueActivePartitions: 10

# 缓存优化
frontend.cacheMaxSizeBytes: 500000000  # 500 MB
frontend.cacheTTLSeconds: 60
history.historyCacheSize: 10000
history.historyCacheTTL: 300
matching.historyCacheSize: 10000

# 长轮询优化
matching.longPollExpirationInterval: 2000000000  # 20秒
frontend.longPollExpirationInterval: 2000000000
```

### 2. 数据库连接池配置

```yaml
persistence:
  datastores:
    default:
      sql:
        maxConns: 100         # 生产环境可以设 100-200
        maxIdleConns: 50      # 生产环境可以设 50-100
        maxConnLifetime: 1h
    visibility:
      sql:
        maxConns: 100
        maxIdleConns: 50
        maxConnLifetime: 1h
```

## Namespace Retention 设置

### 1. 部署时自动创建 (默认 1年)

在 `temporal-simple.yaml` 中已经配置好：
```yaml
namespaces:
  create: true
  namespace:
    - name: default
      retention: 8760h  # 1年
```

还有对应的 Namespace Job，部署时自动创建！

### 2. 用 Wrapper 脚本创建 (推荐本地用)

**必须用 wrapper 脚本，才会自动加 `--retention 8760h`！**

Wrapper 脚本位置：`temporal-wrapper.sh`

**在 Mac 上使用：**
```bash
# 下载脚本
scp root@你的服务器:/root/temporal-deploy/temporal-wrapper.sh ~/bin/temporal
chmod +x ~/bin/temporal

# 确保 ~/bin 在 PATH 最前面
echo 'export PATH="$HOME/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**使用：**
```bash
temporal operator namespace create my-namespace --address localhost:7233
```
自动添加 `--retention 8760h0m0s`！

### 3. 手动创建 (必须加参数!)

```bash
kubectl exec -n temporal-simple deploy/temporal-simple-admintools -- \
  temporal operator namespace create my-namespace \
  --address temporal-simple-frontend:7233 \
  --retention 8760h0m0s
```

### 4. 更新已有 Namespace 的 Retention

```bash
kubectl exec -n temporal-simple deploy/temporal-simple-admintools -- \
  temporal operator namespace update my-namespace \
  --address temporal-simple-frontend:7233 \
  --retention 8760h0m0s
```

## 动态配置说明

### 动态配置文件位置

ConfigMap: `temporal-simple-config`

Key: `dynamic_config.yaml`

### 查看当前动态配置

```bash
kubectl get configmap temporal-simple-config -n temporal-simple -o jsonpath='{.data.dynamic_config.yaml}'
```

### 更新动态配置

1. 编辑 `temporal-simple.yaml` 里的 `dynamic_config.yaml`
2. 重新部署：
   ```bash
   kubectl apply -f temporal-simple.yaml
   kubectl rollout restart deployment/temporal-simple-server -n temporal-simple
   ```

## 开发 → 生产升级

### 1. 只改副本数！

开发环境 (1副本)：
```yaml
spec:
  replicas: 1
```

生产环境 (3-5副本)：
```yaml
spec:
  replicas: 3
```

### 2. MySQL 升级为生产用

- 主从复制
- 或者 MySQL Cluster
- 或者用云厂商的 MySQL
