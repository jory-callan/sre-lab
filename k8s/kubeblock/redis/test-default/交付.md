# 交付 — Redis 测试实例

## 架构

```
                                  ┌──────────────┐
                                  │  Redis Sentinel│
                                  │  (3 副本)     │
                                  └──────┬───────┘
                                         │
          ┌──────────────────────────────┼──────────────────────┐
          │                              │                      │
          ▼                              ▼                      ▼
  ┌──────────────┐              ┌──────────────┐      ┌──────────────┐
  │  Redis Primary│             │  Redis Replica│     │  Redis Replica│
  │  (读写)      │             │  (只读)      │     │  (只读)      │
  └──────────────┘              └──────────────┘      └──────────────┘
```

## 访问方式

| 方式 | 说明 |
|------|------|
| `redis-redis.<ns>` | 主节点 Service（读写） |
| `redis-redis-<idx>.redis-redis-headless.<ns>` | 各 Pod 直连 |
| `redis-redis-sentinel.<ns>` | Sentinel Service |

## 存储

- **Redis data:** 5Gi PVC，默认 StorageClass
- **Sentinel data:** 1Gi PVC，默认 StorageClass

## 账号

| 用户 | 密码 | 说明 |
|------|------|------|
| default | redis@czw123 | 默认用户（ACL 认证） |

查看密码:
```bash
kubectl -n redis get secret redis-account -o jsonpath='{.data.password}' | base64 -d
```

## 注意事项

1. **KubeBlocks 管理** — 请勿手动修改 Pod/Service，通过 Cluster CR 变更
2. **故障切换** — Sentinel 自动处理，KubeBlocks 会更新 Service 路由
3. **密码** — 预配置为 `redis@czw123`，通过 `secret-account.yaml` 管理
4. **持久化** — RDB/AOF 文件保存在 PVC（nfs-client, 5Gi）中
5. **配置** — 默认配置通过 KubeBlocks ConfigMap 管理，如需修改可通过 KubeBlocks 配置功能
