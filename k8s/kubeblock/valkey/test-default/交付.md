# 交付 — Valkey 测试实例

## 架构

```
                                  ┌──────────────┐
                                  │ Valkey Sentinel│
                                  │  (3 副本)     │
                                  └──────┬───────┘
                                         │
          ┌──────────────────────────────┼──────────────────────┐
          │                              │                      │
          ▼                              ▼                      ▼
  ┌──────────────┐              ┌──────────────┐      ┌──────────────┐
  │ Valkey Primary│             │Valkey Replica│     │Valkey Replica│
  │  (读写)      │             │  (只读)      │     │  (只读)      │
  └──────────────┘              └──────────────┘      └──────────────┘
```

## 访问方式

| 方式 | 说明 |
|------|------|
| `valkey-valkey.<ns>` | 主节点 Service（读写） |
| `valkey-valkey-<idx>.valkey-valkey-headless.<ns>` | 各 Pod 直连 |
| `valkey-valkey-sentinel.<ns>` | Sentinel Service |

## 存储

- **Valkey data:** 5Gi PVC，默认 StorageClass
- **Sentinel data:** 1Gi PVC，默认 StorageClass

## 注意事项

1. **KubeBlocks 管理** — 请勿手动修改 Pod/Service，通过 Cluster CR 变更
2. **故障切换** — Sentinel 自动处理，KubeBlocks 会更新 Service 路由
3. **密码** — 默认无密码，如需认证可通过 KubeBlocks 配置
4. **持久化** — RDB/AOF 文件保存在 PVC 中
5. **Valkey = Redis 替代品** — 完全兼容 Redis 协议，现有客户端无需修改
