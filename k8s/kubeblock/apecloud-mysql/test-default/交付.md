# 交付 — ApeCloud MySQL 测试实例

## 架构

```
         ┌──────────────────────┐
         │  ApeCloud MySQL      │
         │  Raft Consensus      │
         │  (3 节点)            │
         └──────┬───────┬───────┘
                │       │
        ┌───────┴──┐ ┌──┴───────┐
        │  MySQL-0  │ │  MySQL-1  │ │  MySQL-2  │
        │  (Leader) │ │ (Follower)│ │ (Follower)│
        └──────────┘ └──────────┘ └──────────┘
```

## 访问方式

| 方式 | 说明 |
|------|------|
| `apecloud-mysql-mysql.<ns>` | 读写 Service（指向 Leader） |
| `apecloud-mysql-mysql-<idx>.apecloud-mysql-mysql-headless.<ns>` | 各 Pod 直连 |

- **MySQL data:** 10Gi PVC，`nfs-client` StorageClass
 
 ## 账号

| 用户 | 密码 | 说明 |

| root | root@czw123 | 管理员账号 |

查看密码:
```bash
kubectl -n mysql get secret mysql-account-root -o jsonpath='{.data.password}' | base64 -d
```

## 注意事项

1. **KubeBlocks 管理** — 请勿手动修改 Pod/Service，通过 Cluster CR 变更
2. **Raft 高可用** — 自动故障切换，最多容忍 1 节点故障
3. **扩容** — 修改 Cluster CR 中 replicas 字段
4. **备份** — 通过 KubeBlocks backup 功能
5. **密码** — 预配置为 `root@czw123`，通过 `secret-account.yaml` 管理
6. **存储** — PVC 使用 `nfs-client` StorageClass，10Gi
7. **配置** — 默认配置通过 KubeBlocks ConfigMap 管理，如需修改可通过 KubeBlocks 配置功能
