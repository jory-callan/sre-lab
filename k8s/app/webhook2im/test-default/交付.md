# 交付 — webhook2im 测试实例

## 访问方式

| 方式 | 地址 | 用途 |
|------|------|------|
| HTTPS | https://webhook2im.czw-sre.internal | Webhook 接收端 |

## 配置说明

应用通过 ConfigMap 挂载 `config.yaml` 配置，包含路由表和输出目标。

当前配置的路由：

| 路径 | 来源 | 输出 |
|------|------|------|
| /alertmanager | AlertManager 告警 | 飞书文本消息 |

如需修改配置，编辑 `values.yaml` 中的 `config.data` 后执行 `bash install.sh install` 即可。

## 发送测试 Webhook

```bash
# 健康检查
curl -k https://webhook2im.czw-sre.internal/health

# 模拟 AlertManager 告警
curl -X POST https://webhook2im.czw-sre.internal/alertmanager \
  -H "Content-Type: application/json" \
  -d '{
    "status": "firing",
    "alerts": [{
      "labels": {"alertname": "CPU高", "severity": "critical"},
      "annotations": {"description": "CPU 使用率超过 90%"},
      "startsAt": "2025-01-01T00:00:00Z"
    }]
  }'
```

## 注意事项

- 飞书 webhook URL 在 `config.data` 中配置，如需修改请更新 values.yaml
- 默认 1 副本，如需 HA 请调整 replicaCount 并启用 PDB
