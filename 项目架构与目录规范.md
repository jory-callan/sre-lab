# 项目架构与目录规范

## 核心原则

- 80/20：先做当前真正需要的，不搞过度设计
- 简单直接：不搞复杂统一入口，每个方案独立完整
- 层级清晰：01-physical → 02-platforms → 03-infrastructure → 04-applications


## 目录结构

```
<project-root>/
├── 01-physical/                    # 物理层：裸机、基础系统配置
├── 02-platforms/                   # 平台层：Docker、K3s、Kubernetes
├── 03-infra-docker/                # 基础设施 Docker Compose 应用
├── 03-infra-k8s/                   # 基础设施 K8s 应用
├── 04-apps-docker/                 # 业务应用 Docker Compose
├── 04-apps-k8s/                    # 业务应用 K8s
├── 99-archive/                     # 归档旧文件
├── AGENT.md                        # AI 助手规则（精简，纯文本）
└── README.md                       # 项目说明
```


## K8s 应用目录规范

### 目录结构
```
<component-dir>/
├── manifests/                      # K8s 原生清单（必备）
│   ├── remote-<name>-<version>.yaml    # 远程下载的原始文件，用 remote- 前缀
│   ├── <config-name>.yaml              # 自己写的配置文件，带详细注释
│   └── ...
├── helm/                           # Helm Chart（可选）
│   ├── remote-<chart-name>.tgz         # 远程下载的 Chart，用 remote- 前缀
│   ├── values-dev.yaml                 # Dev 环境 Values
│   ├── values-prod.yaml                # Prod 环境 Values
│   └── ...
├── kustomize/                      # Kustomize（可选）
│   ├── base/
│   ├── overlays/dev/
│   └── overlays/prod/
├── install.sh                      # 安装脚本（只有一个，默认方式）
├── uninstall.sh                    # 卸载脚本（只有一个）
├── README.md                       # 简单说明：只讲怎么用
└── 高级配置指南.md                 # 详细高级玩法（可选）
```

### 命名约定

#### 远程资源
- 格式：`remote-<name>-<version>.yaml`
- 示例：`remote-metallb-v0.14.8.yaml`、`remote-ingress-nginx-v1.12.0.yaml`

#### 配置文件
- 自己写的配置文件，用有意义的名字
- 示例：`ipaddresspool.yaml`、`l2advertisement.yaml`
- **必须带详细注释**，说明配置项含义

#### 环境区分
- 不搞 dev/prod 文件夹层级
- 大多数应用不需要区分 dev/prod，直接一个配置文件
- 确需区分的，通过文件名后缀：`values-dev.yaml`、`values-prod.yaml`

## Docker Compose 应用目录规范

### 目录结构
```
<app-dir>/
├── docker-compose.yaml            # 主要配置（可选 dev/prod 后缀）
├── docker-compose-dev.yaml        # Dev 环境（可选）
├── docker-compose-prod.yaml       # Prod 环境（可选）
├── config/                        # 配置文件（可选）
├── data/                          # 数据目录（gitignore）
├── install.sh                     # 安装脚本（可选）
├── uninstall.sh                   # 卸载脚本（可选）
└── README.md                      # 简单说明：只讲怎么用
```

### README.md 要求

**只讲怎么用，不列出文件树**，简单为主：
```
# <应用名>

## 快速开始
```bash
docker-compose up -d
```

## 验证
```bash

```

```bash

```
## 域名规范

- 避免使用 `*.local` 域名（macOS 特殊处理，会有 5 秒延迟）
- 统一使用 `*.czw-sre.internal` 域名


## Git 提交规范

- 格式：`type(scope): subject`
- 示例：`feat(metallb): add advanced config guide`
- 在提交信息末尾，附工具版本号和使用的模型
